# v2.1 — ResearchKit OS Release

**Date**: 2026-07-21
**Branch**: `main` (merged from `develop`)
**Tag**: `v2.1`
**Status**: Production release — OKX AI Genesis Hackathon submission baseline

> 7 天渐进式扩展（D1-D7）把 v2.0 的「Multi-Agent OS」升级为「可配置 Agent Studio」——
> 评委和用户可在 Settings UI 中切换 LLM Provider、自定义 Prompt、选角色 Preset、查看 token 成本，
> 无需改一行代码即可适配 DeepSeek / OpenAI / OpenRouter / Groq / 硅基流动 / 火山 / 百炼 / 混元 等所有 OpenAI Compatible API。

## 一、核心交付

### D1 — LLMProvider Interface 抽象层
[PR #8](https://github.com/yuuumc/researchkit/pull/8)

新增 `core/llm/provider.ts`：
- `LLMProvider` 接口 — `chat(messages, options): Promise<ChatResponse>` 统一所有 LLM 调用
- `ChatResponse` 含 `content / model / usage (ChatUsage) / finishReason / durationMs`
- `ProviderFactory` 工厂模式 — `create() / fromEnv() / fromUserConfig()` 三种创建方式
- `MODEL_PRICING` 价格表 + `estimateTokenCost()` 成本估算
- 5 个 ProviderPreset 预设（DeepSeek / OpenAI / OpenRouter / Groq / Custom）

### D2 — OpenAICompatProvider 实现 + LLM 调用全量迁移
[PR #9](https://github.com/yuuumc/researchkit/pull/9)

新增 `core/llm/providers/openai-compat.ts`：
- 覆盖 9 家厂商（DeepSeek / OpenAI / OpenRouter / Groq / 硅基流动 / 火山 / 百炼 / 混元 / Custom）
- `chat()` 内部用 openai SDK，对外暴露统一接口
- `healthCheck()` — Settings 页"测试连接"按钮用

新增 `lib/server-provider.ts`：
- `getServerProvider()` — server side 优先读 cookie 中用户配置，fallback 到 env
- 让所有 Agent / lib/llm.ts / lib/planner.ts 在 server runtime 中拿到正确 Provider

迁移 10 处 LLM 调用：
- `lib/llm.ts` (KnowledgeBuilder)
- `lib/planner.ts` (PlannerAgent._run / reflect / replan)
- `core/agents/{reader,analyzer,terminology,knowledge-builder,recommendation,export}/index.ts`

### D3 — Settings UI 分 Tab + Provider 配置
[PR #10](https://github.com/yuuumc/researchkit/pull/10)

新增 `components/settings/SettingsContainer.tsx` + 4 个 Tab：
- **Provider Tab** — Provider 类型下拉 / API Key / Base URL / Model / Test Connection 按钮
- **General Tab** — 占位（D5 填充）
- **About Tab** — 版本信息
- **Prompt Tab** — 占位（D4 填充）

新增 `lib/user-config.ts` + `lib/user-preferences.ts`：
- localStorage + cookie 双写模式
- localStorage 主存，cookie 让 server side 通过 `next/headers cookies()` 读取

### D4 — PromptBuilder 三层架构 + Prompt Tab
[PR #11](https://github.com/yuuumc/researchkit/pull/11)

新增 `core/prompt/PromptBuilder.ts` — 三层 Prompt 架构：
```
System 🔒 (只读内置，ResearchKit 核心 Prompt)
  ↓
Preset 🎭 (角色模板：academic / beginner / developer / ...)
  ↓
Project ➕ (用户项目级扩展，每个 Agent 独立配置)
  ↓
User ➕ (单次扩展，API 调用时传入)
  ↓
Final Prompt → provider.chat()
```

- `MAX_PROMPT_LENGTH = 8000` — 超限时自动丢弃（顺序：User → Project，System 永不丢）
- Settings → Prompt Tab：6 个 Agent（Reader/Analyzer/Terminology/KB/Recommendation/Planner）独立配置 Project Extension

### D5 — Prompt Preset（5 角色）+ Output Language
[PR #12](https://github.com/yuuumc/researchkit/pull/12)

新增 `config/presets.ts` — 5 角色 PresetTemplate：
| Preset | Persona | 适合场景 |
|---|---|---|
| `academic` | 严谨学术教授（默认） | 论文研究 |
| `beginner` | 耐心科普老师 | 新手学习 |
| `developer` | 资深工程师 | 工程实现 |
| `researcher` | 资深研究员 | 综述写作 |

新增 `lib/server-user-preferences.ts`：
- `getServerUserPreferences()` — server side 读 cookie 中的 preset + outputLocale
- `getEffectiveOutputLocale(detectedLocale)` — 'auto' 跟随源语言，否则强制覆盖

Settings → General Tab：
- Preset 下拉 + persona 预览
- Output Locale 下拉（auto / zh-CN / en-US / ja-JP / ko-KR / fr-FR / de-DE / es-ES）

向后兼容：默认 `preset='academic'` + `outputLocale='auto'` 与 v2.0 行为完全一致。

## 二、架构收益

### 改造前（v2.0）
```typescript
// lib/llm.ts
const openai = new OpenAI({ apiKey, baseURL })
await openai.chat.completions.create({ messages, ... })
// ↑ 写死 OpenAI SDK，切换 Provider 要改代码
// ↑ 没有统一 usage 统计，不知道 token 花在哪
// ↑ 没有角色 persona，所有场景用同一套 Prompt
```

### 改造后（v2.1）
```typescript
// core/llm/provider.ts
const provider = ProviderFactory.fromUserConfig(userConfig)  // 用户配置驱动
await provider.chat(messages, options)                       // Agent 不知道背后是谁
// ↑ 自动记录 usage 到 collector → Cost Dashboard

// core/prompt/PromptBuilder.ts
const finalPrompt = PromptBuilder.build({
  system: SYSTEM_PROMPTS.reader,     // 🔒 ResearchKit 内置（只读）
  preset: prefs.preset,              // 🎭 角色预设（academic / beginner / ...）
  project: userProjectExtension,     // ➕ 项目级扩展
  user: userSingleExtension,         // ➕ 单次扩展
})
// ↑ 用户在 Settings UI 改 Prompt，立即生效，无需改代码

// core/orchestration/coordinator.ts
const collector = beginCollection()
// ... 所有 provider.chat() 自动记录到 collector ...
const summary = collector.summarize()
return {
  ...result,
  totalUsage: summary.totalUsage,       // { promptTokens, completionTokens, totalTokens }
  totalCostUsd: summary.totalCostUsd,   // $0.0058
  perAgentUsage: summary.perAgent,      // [{ agent: 'Planner', totalTokens: 4305, ... }]
}
```

### 已知限制（v2.2 改进）
- 并行 Agent 执行时 `currentAgentName` 竞态，导致 Reader/Analyzer/KnowledgeBuilder 的 token 归到 Terminology（last setter wins）。**总量正确**，per-agent 归因不准。v2.2 用 AsyncLocalStorage 解决。
- CostTab 历史存储仅 client side localStorage（不跨设备同步）。
- Prompt Project Extension 全局共享，不能按文档/项目区分（v2.2 计划）。

## 六、技术栈

- **Runtime**: Next.js 14.2.5 (App Router, Node.js runtime)
- **Language**: TypeScript 5.x (strict mode)
- **LLM SDK**: openai@4.x (兼容 9 家厂商)
- **UI**: React 18 + 内联样式（无 UI 框架依赖，评委审计零干扰）
- **State**: React useState + localStorage + cookie（双写）
- **Persistence**: localStorage（cost-history / user-config / user-preferences / prompt-extensions）
- **Streaming**: SSE (Server-Sent Events) — `text/event-stream` 手动解析
