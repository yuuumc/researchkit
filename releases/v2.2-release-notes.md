### D12 — Plugin System 起步（三层架构）
[PR #18](https://github.com/yuuumc/researchkit/pull/18)

新增 `types/plugin.ts`（216 行）+ `core/plugins/registry.ts`（84 行）+ `core/plugins/sample-plugins.ts`（260 行）+ `lib/plugin-states.ts`（107 行）+ `components/PluginPanel.tsx`：

**三层架构**：
```
UI (PluginPanel) ──→ Registry (singleton) ──→ Plugins (ExportPlugin interface)
```

**ExportPlugin 接口**：
- `meta` — id / name / version / description / tags / requiresConfig / configSchema
- `capabilities` — export / publish / download
- `validate?(kc)` — 预校验
- `export(ctx)` — 执行导出

**热插拔**：插件可在运行时 `register/unregister`，状态持久化到 localStorage。
**幂等**：相同输入多次调用结果一致。
**不 throw**：所有错误通过 `ExportResult.success=false` 返回。

**2 个示例插件**：
- `jsonDownloadPlugin` — KC → JSON 文件（含 `__meta` 元数据）
- `markdownDownloadPlugin` — KC → Markdown 文件（含 YAML frontmatter）

### D13 — Onchain Export Plugin（X Layer 链上锚定）
[PR #19](https://github.com/yuuumc/researchkit/pull/19)

新增 `lib/onchain-utils.ts`（189 行）+ `lib/onchain-ledger.ts`（118 行）+ `core/plugins/onchain-export.ts`（200 行）：

**Demo Mode 设计**（明确标注，不欺瞒评审）：
- ✅ **真实 SHA-256**：用 Web Crypto API `crypto.subtle.digest('SHA-256', ...)` 真实计算
- ✅ **确定性派生**：同 KC + 同 wallet + 同 chainId → 同 tx hash
- ⚠️ **Mock tx hash**：EVM 兼容格式（64 hex chars），未实际广播到 X Layer mainnet
- ⚠️ **localStorage ledger**：FIFO 50 条上限，仅本地可见

**核心函数**：
- `sha256(text)` — Web Crypto API 真实计算（带 fallbackHash 兜底）
- `deriveTxHash({kcSha256, walletAddress, nonce, chainId})` — 确定性派生 64 hex chars
- `deriveKcTokenId(kcSha256)` — 派生 token ID（1M-10M 范围）
- `deriveBlockNumber(timestamp)` — 基于时间戳模拟 X Layer block height
- `buildExplorerTxUrl/buildExplorerAddressUrl` — OKLink explorer URL

**OnchainExportPlugin 实现**：
- 真实计算 KC SHA-256（`canonicalizeKc` 字段排序保证稳定性）
- 防重发布（同 KC+wallet+chain → 返回已有记录）
- 模拟网络延迟（800-1300ms）
- 版本号 `0.9.0-mvp` + 标签 `experimental/demo`

**关键修复**：BigInt literals 在 ES2019 target 下不可用，改用 `Math.imul` + `>>> 0` 实现 32-bit hash。

**UI 增强**：`PluginPanel.tsx` +295 行
- 注册 onchainExportPlugin 到 BUILTIN_PLUGINS
- `OnchainHistory` 子组件 — 可折叠历史发布记录列表
- `OnchainRecordItem` — 单条记录卡片（tx hash + block + gas + wallet + 详情展开）
- 底部 Demo Mode 警告条（橙色）

## 二、架构收益

### 改造前（v2.1）
- KC 是静态文档：生成后只能导出 Markdown / JSON
- 无对比能力：用户手动对比两篇论文
- 无对话能力：KC 不能追问
- 无解释能力：所有受众看同一份内容
- 无扩展能力：导出格式写死在代码里
- 无链上锚定：KC 无法持久化到区块链

### 改造后（v2.2）
```typescript
// D9 — Smart Suggestion（纯客户端，零成本）
const suggestions = findSimilarKCs(currentKC, history)  // 启发式评分
if (suggestions[0]?.score >= 30) {
  showBanner(suggestions[0])  // amber banner
}

// D10 — Chat with KC（注入完整上下文）
const messages = [
  { role: 'system', content: buildKcContext(knowledgeCard) },
  ...history,
  { role: 'user', content: question },
]
const reply = await provider.chat(messages, { temperature: 0.4 })

// D11 — Explain for Audience（4 种受众）
const audience: Audience = 'high_school'  // 或 software_engineer / researcher / product_manager
const explanation = await provider.chat([
  { role: 'system', content: AUDIENCES[audience].focus },
  { role: 'user', content: kc.summary },
], { temperature: 0.5 })

// D12 — Plugin System（热插拔）
interface ExportPlugin {
  meta: PluginMeta
  capabilities: PluginCapability[]
  validate?(kc: KnowledgeCard): string | null
  export(ctx: PluginContext): Promise<ExportResult>
}
registry.register(jsonDownloadPlugin)
registry.register(markdownDownloadPlugin)
registry.register(onchainExportPlugin)

// D13 — Onchain Export（真实 SHA-256 + mock tx hash）
const sha = await sha256(canonicalizeKc(knowledgeCard))  // 真实 Web Crypto
const txHash = deriveTxHash({ kcSha256: sha, walletAddress, nonce, chainId })  // 确定性派生
const record: OnchainRecord = {
  kcSha256: sha,
  txHash,
  blockNumber: deriveBlockNumber(Date.now()),
  tokenId: deriveKcTokenId(sha),
  // ...
}
appendRecord(record)  // localStorage ledger

// D14 — Prompt Playground（双栏 + 4 preset）
const result = await provider.chat([
  { role: 'system', content: systemPrompt },
  { role: 'user', content: userPrompt },
], { temperature, maxTokens, responseFormat })
```

## 六、技术栈

- **Runtime**: Next.js 14.2.5 (App Router, Node.js runtime)
- **Language**: TypeScript 5.x (strict mode)
- **LLM SDK**: openai@4.x（兼容 9 家厂商，继承自 v2.1）
- **UI**: React 18 + 内联样式（无 UI 框架依赖，评委审计零干扰）
- **State**: React useState + localStorage（plugin-states / onchain-ledger / cost-history）
- **Streaming**: SSE (Server-Sent Events) — `text/event-stream` 手动解析
- **Crypto**: Web Crypto API（`crypto.subtle.digest` 真实 SHA-256）
