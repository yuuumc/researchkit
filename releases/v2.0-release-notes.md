## 核心升级

### 1. 统一 AgentInterface
所有 Agent 实现统一接口：
```typescript
export interface AgentInterface {
  name: string
  description: string
  capabilities: AgentCapability[]
  execute(ctx: AgentContext): Promise<AgentResult>   // v2.0 统一入口
  handleMessage(message: AgentMessage): Promise<AgentMessage>  // @deprecated v1.0 兼容
  getCapabilities(): AgentCapability[]
}
```

新增 Agent（如 `CriticAgent`）只需 `class CriticAgent implements AgentInterface` + 注册到 `AGENTS` 表，executor 和 coordinator 完全不用改 — **真正的开放-封闭原则**。

### 2. 统一 AgentContext
所有 Agent 共享同一份上下文，未来加 Memory / UserPref / History 不用改接口：
```typescript
export interface AgentContext {
  document: { content, language, locale, title?, source?, metadata? }
  workflow: { inputType, requiredSchema, complexity, plan? }
  previous: { reader?, analyzer?, terminology?, knowledgeCard?, recommendation? }
  options: { locale, sourceLocale, languageDirective, detailLevel? }
  promptPatch?: { focus_fields, ignore_fields, extra_instruction }
}
```

### 3. 模块化目录结构
```
core/
├── agents/{reader,analyzer,terminology,knowledge-builder,recommendation,export}/
│   └── index.ts                    # class XxxAgent implements AgentInterface
├── orchestration/
│   ├── planner.ts                  # runPlanner + fallbackPlan
│   ├── executor.ts                 # AGENTS 注册表 + callAgent + buildAgentContext
│   ├── workflow.ts                 # runReflectionLoop + extractKnowledgeCard
│   └── coordinator.ts              # 薄壳：locale → plan → execute → workflow → export
types/
├── agent.ts                        # AgentInterface + AgentContext + AgentResult
├── knowledge.ts                    # ReaderOutput / AnalyzerOutput / KnowledgeCard ...
├── workflow.ts                     # Plan / Step / Trace / Iteration
└── export.ts                       # ExportOutput
prompts/
├── reader.ts / analyzer.ts / terminology.ts / recommendation.ts / planner.ts
```

## 架构对比

### v1.0（重构前）
- 10 个核心文件 × 每个 500 行 = 找代码困难
- Agent 是对象字面量，新增需改 coordinator
- 类型散落各文件，IDE 无智能提示

### v2.0（重构后）
- 50 个文件 × 每个 50~150 行 = 开发速度更快
- Agent 是 class，注册到 AGENTS 表即用
- 类型统一在 types/，IDE 智能提示完备
