D31 — Plugin System v2 (1/3): 接口扩展 + 生命周期 + 权限模型

Phase 5 Day 1 — 为 Plugin 系统加入 category 分类、生命周期钩子、
权限声明三大 v2 接口，UI 上展示 category badge 和可折叠权限详情。

- types/plugin.ts:
  - 新增 PluginCategory ('export' | 'source' | 'sync')
  - 新增 PluginLifecycle 接口 (onInstall/onEnable/onDisable/onUninstall)
  - 新增 PluginPermissions 接口 (kcFields/externalApis/network/
    filesystem/walletSignature)
  - PluginMeta 加 category? 字段
  - ExportPlugin 加 lifecycle? + permissions? 可选字段

- core/plugins/registry.ts:
  - 新增 listByCategory(cat) — 按 category 过滤
  - 新增 triggerLifecycle(id, event, ctx) — 调用生命周期钩子
    (异常捕获，失败不抛出)
  - 新增 getPermissions(id) — 查询权限声明

- core/plugins/sample-plugins.ts:
  - jsonDownloadPlugin / markdownDownloadPlugin 加 category='export'
    + permissions + lifecycle (onInstall/onUninstall demo)

- core/plugins/onchain-export.ts:
  - 加 category='export' + permissions (network/walletSignature=true)
  - 加 lifecycle onEnable (校验钱包地址格式) + onUninstall

- components/PluginPanel.tsx:
  - mount 时把 BUILTIN_PLUGINS 注册到 pluginRegistry
  - handleToggleEnabled 改为 async，onEnable 失败时回滚 toggle
  - PluginCard 加 category badge (按颜色区分 export/source/sync)
  - PluginCard 加 🔒 perms 按钮，点击展开权限详情面板
    (KC 字段/外部 API/网络/文件系统/钱包签名)
