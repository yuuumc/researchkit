### D1-D4 架构收益

改造前：
```typescript
// lib/llm.ts
const openai = new OpenAI({ apiKey, baseURL })
await openai.chat.completions.create({ messages, ... })
```

改造后：
```typescript
// core/llm/provider.ts
const provider = ProviderFactory.create(userConfig)  // 用户配置驱动
await provider.chat(messages, options)                // Agent 不知道背后是谁

// core/prompt/PromptBuilder.ts
const finalPrompt = PromptBuilder.build({
  system: SYSTEM_PROMPTS.reader,     // 🔒 ResearchKit 内置（只读）
  project: userProjectExtension,     // ➕ 项目级扩展
  user: userSingleExtension,         // ➕ 单次扩展
  preset: 'academic',                // 🎭 角色预设
})
```

### Onchain Export Plugin 设计（替代原 OnchainExportAgent）

**优化理由**（用户建议）：Onchain 只是 Exporter 之一，不应独占一个 Agent 槽位。改为 Export Plugin 架构更统一。

```typescript
// plugins/exporters/onchain/index.ts
export class OnchainExporter implements ExportPlugin {
  name = 'onchain'
  description = '把知识卡内容 hash 写到 X Layer'
  supportedFormats = ['json'] as const

  async export(knowledgeCard: KnowledgeCard, options: OnchainOptions): Promise<ExportResult> {
    // 1. 计算 knowledgeCard 的 SHA-256 hash
    // 2. 调用 X Layer SDK 写入链上
    // 3. 返回 { txHash, blockNumber, explorerUrl }
  }
}

// plugins/exporters/index.ts（注册表）
const EXPORTERS: Record<string, ExportPlugin> = {
  markdown: new MarkdownExporter(),
  obsidian: new ObsidianExporter(),
  json: new JsonExporter(),
  yaml: new YamlExporter(),
  onchain: new OnchainExporter(),  // 新增
}

// Export Agent 调用所有已注册的 Exporter（零改动）
// 新增 Exporter 只需注册到 EXPORTERS 表
```

| # | 任务 | 验证指标 |

| 1 | **英文论文质量专项优化** | 50 篇英文论文测试，作者/年份/领域准确率 ≥ 95% |

| 7 | **UI 细节与交互打磨** | Loading 状态 / Error 边界 / 响应式布局 / 移动端 |
| 8 | **错误处理与降级策略** | 单个 Agent 失败不影响整体 Pipeline，降级到 fallback 输出 |
| 9 | **文档完善** | 用户手册 / API 文档 / 架构图 / 部署指南 |

## 九、Plugin System 设计（v2.2 起步）

**核心理念**（用户建议第 7 条）：现在已经开始插件化，为什么不继续？

```
plugins/
├── providers/              # LLM Provider 插件
│   ├── openai-compat/
│   ├── deepseek/
│   ├── openrouter/
│   └── groq/
├── agents/                 # Agent 插件
│   ├── reader/
│   ├── analyzer/
│   ├── explain/            # v2.2 新增
│   └── custom/             # 用户自定义 Agent
├── exporters/              # Export 插件
│   ├── markdown/
│   ├── obsidian/
│   ├── json/
│   ├── yaml/
│   └── onchain/            # v2.2 新增
└── tools/                  # Tool 插件
    ├── web-search/
    ├── arxiv-api/
    └── semantic-scholar/
```

**接口定义**：
```typescript
// plugins/types.ts
export interface Plugin<T = any> {
  name: string
  version: string
  type: 'provider' | 'agent' | 'exporter' | 'tool'
  init(config: T): Promise<void>
}

export interface ProviderPlugin extends Plugin {
  type: 'provider'
  chat(messages: ChatMessage[], options?: ChatOptions): Promise<string>
}

export interface AgentPlugin extends Plugin {
  type: 'agent'
  execute(ctx: AgentContext): Promise<AgentResult>
}

export interface ExporterPlugin extends Plugin {
  type: 'exporter'
  export(knowledgeCard: KnowledgeCard, options?: any): Promise<ExportResult>
}

export interface ToolPlugin extends Plugin {
  type: 'tool'
  call(input: any): Promise<any>
}
```

## 十、核心架构原则（贯穿所有阶段）

### 1. 模型无关（Model-agnostic）
```
Agent → LLMService → Provider → HTTP Client
```

### 2. 三层 Prompt 拼接（安全可扩展）
```typescript
finalPrompt = [
  systemPrompt,      // 🔒 ResearchKit 内置（只读）
  projectPrompt,     // ➕ Project 级 Extension
  userPrompt,        // ➕ 单次 Extension
].join('\n\n')
```

### 3. 角色驱动而非 Prompt 驱动
```
Reader Style:  ○ Fast  ○ Detailed  ○ Critical  ○ Educational
```

### 4. 开放-封闭原则（v2.0 已验证）
新增 Agent / Provider / Exporter 只需：
1. `class XxxPlugin implements XxxPluginInterface`
2. 注册到对应表（AGENTS / PROVIDERS / EXPORTERS）

### 5. Plugin 化（v2.2 起步）
providers / agents / exporters / tools 全部插件化，新增任何模块都是 Plugin。

### 6. ChainHack Web3 元素
- Onchain Export Plugin 写知识卡 hash 到 X Layer
- 后续 v4.0 可扩展为去中心化知识图谱
- OKX Onchain Points 积分系统对齐

### 7. Settings 分 Tab（防止越来越乱）
6 Tab：General / AI / Agents / Workflow / Export / Developer

## 十六、给新手的核心建议（沿用 v2.0 纪律）

1. **不要追求 Stage 1 全部完成** — 保 D1-D5（Provider + Prompt Builder + Preset）就够 OKX
2. **每天必须冒烟测试** — 不测就提交 = 自掘坟墓
3. **遇到卡壳超 1 小时立刻停** — 切回 main 上一稳定版本，第二天再战
4. **每个 PR 都要能独立合并** — 不要攒多天一起合并
5. **System Prompt 永远锁定只读** — 三份建议的核心共识，破坏 System Prompt = Pipeline 必崩
6. **Provider 抽象不动 Agent 调用方** — `lib/llm.ts` 内部重构，Agent 继续调 `generateKnowledgeCard` / `callLLM`
7. **v2.2.5 不要跳过** — 一个稳定高质量的系统，比多两个半成品功能更有说服力
8. **Settings 必须分 Tab** — 新功能必须归 Tab，防止单页越来越乱
9. **Onchain 不是独立 Agent，是 Export Plugin** — 架构统一性比"看起来多一个 Agent"重要
